# RepoShare 设计系统

状态：第一阶段基础规范  
最后更新：2026-06-19

本文档是 RepoShare 应用壳层的正式设计与实现规范。各套导出模板保持视觉独立，因为模板 CSS 属于最终生成图片的一部分，不受应用壳层主题约束。

文中的“必须”“应该”“可以”分别表示强制要求、推荐方案和可选方案。当本文档与旧组件实现冲突时，新代码以本文档为准，并在后续迁移提交中说明差异。

## 1. 产品设计原则

RepoShare 的核心任务很短：

1. 加载 GitHub 仓库。
2. 选择视觉模板。
3. 调整内容和画布选项。
4. 导出图片。

界面必须始终让预览画布成为视觉中心，编辑控件只在用户需要时出现。

### 布局决策

RepoShare 不应该默认同时显示两个常驻侧栏。当前只有七套模板和少量显示、导出控制项，不值得持续占用预览空间。

目标布局采用**预览主舞台 + 单个上下文面板**：

- 工具轨道用于选择“模板”“内容”或“画布”。
- 同一时间只展示当前工具对应的一个设置面板。
- 切换工具时替换面板内容，不叠加多个面板。
- 关闭设置面板后，空间归还给预览画布。
- 导出是顶部命令栏操作，也是移动端的主要操作。

这一结构借鉴辅助面板（supporting pane）模式，但不照搬大型桌面编辑器。PostSpark 可作为“画布优先”的视觉参考；RepoShare 的功能更少，因此控件密度也必须更低。

## 2. 参考依据

本设计系统借鉴成熟规范的原则，不复制其品牌视觉：

- [GitHub Primer 布局规范](https://primer.style/product/getting-started/foundations/layout/)：强调聚焦内容和 viewport range。Primer 只在 `>=1400px` 时允许可选的第三栏，而不是让所有桌面界面默认三栏。
- [Atlassian 间距规范](https://atlassian.design/foundations/spacing)：使用 8px 基础单位和受约束的间距刻度。
- [Atlassian 字体规范](https://atlassian.design/foundations/typography/product-typefaces-and-scale)：产品界面采用紧凑字号，并让行高与间距体系对齐。
- [Material 3 自适应布局](https://m3.material.io/foundations/adaptive-design/canonical-layouts)：辅助面板应随可用宽度停靠、覆盖或收起。
- [WCAG 2.2 目标尺寸](https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html)：指针目标至少为 24x24 CSS 像素；RepoShare 的主要触摸操作统一使用 44px。
- [WCAG 非文本对比度](https://www.w3.org/WAI/WCAG22/Understanding/non-text-contrast.html)：有意义的控件边界和状态至少需要 3:1 对比度。

### Liquid Glass 实现

应用控制层使用 [`@creativoma/liquid-glass`](https://github.com/creativoma/liquid-glass) 提供的现成 React 组件，不自行实现 SVG 折射算法。该库使用 MIT 协议，支持 React 18/19，并为 Safari/iOS 提供简化滤镜降级。

- 可以使用：顶部命令栏、宽屏设置面板、覆盖式设置面板、移动端底部工具栏和底部抽屉。
- 禁止使用：预览画布、导出模板、表单每一行、所有按钮逐个包裹。
- 玻璃只表达浮于内容之上的控制层；文字和表单仍需使用足够不透明的语义表面保证可读性。
- 当浏览器不支持完整折射时，必须保留模糊、边框和语义背景，不影响任何操作。

## 3. Token 与代码归属

设计数值的唯一来源：

- `src/styles/tokens.css`：颜色、间距、字体、形状、阴影、动效和层级。
- `src/styles/base.css`、`workspace.css`、`inspector.css`、`responsive.css`：按职责拆分的应用壳层样式。
- `src/lib/design.ts`：响应式范围、应用布局尺寸和导出画布尺寸。

组件必须使用语义变量。应用壳层组件中禁止新增任意颜色值和未登记的间距值。

### 间距刻度

基础单位为 8px。组件内部允许使用二分之一和四分之一刻度。

| Token | 数值 | 主要用途 |
| --- | ---: | --- |
| `--rs-space-0` | 0px | 重置间距 |
| `--rs-space-025` | 2px | 仅用于视觉校正 |
| `--rs-space-050` | 4px | 图标与细节之间的紧凑间距 |
| `--rs-space-075` | 6px | 紧凑菜单内部间距 |
| `--rs-space-100` | 8px | 同组控件间距 |
| `--rs-space-150` | 12px | 控件水平内边距 |
| `--rs-space-200` | 16px | 面板内边距、设置组间距 |
| `--rs-space-250` | 20px | 保留的中间间距 |
| `--rs-space-300` | 24px | 普通画布和页面区域间距 |
| `--rs-space-400` | 32px | 宽屏画布和页面区域间距 |
| `--rs-space-500` | 40px | 大型结构间距 |
| `--rs-space-600` | 48px | 空状态间距 |
| `--rs-space-800` | 64px | 命令栏、移动工具栏尺寸 |
| `--rs-space-1000` | 80px | 特殊的大型留白 |

使用规则：

- 控件中的图标与文字：8px。
- 同一工具栏中的相邻按钮：8px。
- 不同命令组之间：16px。
- 同一个设置组中的控件：8px。
- 设置面板中的不同设置组：16px；只有语义不够清晰时才增加分隔线。
- 普通页面区域：24px；宽屏页面区域：32px。
- 使用负间距或 Tailwind 任意值前，必须先更新设计系统。

### 字体规范

| 用途 | 字号 / 行高 | 字重 |
| --- | --- | --- |
| 辅助文字、分组标题 | 12 / 16px | 500 或 600 |
| 默认产品文字 | 14 / 20px | 400 或 500 |
| 强调正文、紧凑标题 | 16 / 24px | 600 |
| 页面或面板标题 | 20 / 28px | 600 或 700 |

字距始终为 `0`。应用壳层使用系统无衬线字体。模板中的装饰字体不得影响应用界面。

### 形状与阴影

| 用途 | 数值 |
| --- | --- |
| 小圆角 | 4px |
| 中圆角 | 6px |
| 普通面板最大圆角 | 8px |
| 悬浮玻璃控制层圆角 | 16px |
| 移动端玻璃浮层圆角 | 20px |
| 默认边框 | 1px |
| 焦点指示器 | 2px |

普通文字标签不得使用胶囊形容器。胶囊形状只用于标签、状态和分段选择。命令栏、工具轨道、设置面板、底部工具栏和底部抽屉属于悬浮玻璃控制层，可以使用 16–20px 圆角和外部留白；内容区域不能全部设计成漂浮卡片，也不能在设置面板内嵌套卡片。阴影只表达真实层级：抬升控件、浮层、Sheet 和预览边框。

## 4. 色彩与主题

应用代码使用以下语义角色：

- 画布：`--rs-color-canvas`、`--rs-color-canvas-grid`
- 表面：`surface`、`surface-raised`、`surface-hover`、`surface-active`、`surface-selected`、`surface-disabled`
- 文字：`text`、`text-secondary`、`text-muted`、`text-disabled`、`text-inverse`
- 边框：`border`、`border-strong`、`border-disabled`
- 强调色：`accent`、`accent-hover`、`accent-active`、`accent-selected`、`accent-disabled`
- 危险状态：`danger`、`danger-hover`、`danger-active`、`danger-surface`
- 浮层与焦点：`overlay`、`focus`

浅色和深色主题都必须完整定义所有语义角色。深色模式不能通过反色滤镜实现，也不能使用大面积纯黑作为主要表面。强调色只用于主操作、选中、焦点和进度。

### 组件状态矩阵

| 状态 | 表面 | 边框 | 文字或图标 | 额外规则 |
| --- | --- | --- | --- | --- |
| 默认 | `surface` | `border` | `text-secondary` | 不使用多余阴影 |
| 悬停 | `surface-hover` | `border-strong` | `text` | 只适用于指针设备 |
| 按下 | `surface-active` | `border-strong` | `text` | 不能引发布局移动 |
| 选中 | `surface-selected` | `accent-selected` | `text` | 不能只依靠颜色表达 |
| 键盘焦点 | 保持当前状态 | `focus` | 保持当前状态 | 两种主题下均显示 2px 指示器 |
| 禁用 | `surface-disabled` | `border-disabled` | `text-disabled` | 不响应悬停 |
| 危险操作 | `danger-surface` | `danger` | `danger` | 可能丢失数据时需要确认 |

## 5. 响应式布局

响应式范围代表不同的产品模式，不只是 CSS 断点。

| 模式 | 宽度 | 布局行为 |
| --- | --- | --- |
| 紧凑模式（Compact） | 320–767px | 单列预览、底部工具栏、底部抽屉 |
| 常规模式（Regular） | 768–1399px | 预览 + 覆盖式设置面板，不显示常驻侧栏 |
| 宽屏模式（Wide） | >=1400px | 56px 工具轨道、可选的 288px 停靠面板、预览主舞台 |

### 宽屏桌面

```text
+---------------------------------------------------------------------+
| 64px 命令栏：品牌 | 仓库输入 | 主题 | 导出                         |
+------+------------------+--------------------------------------------+
| 56px | 可选 288px       |                                            |
| 工具 | 设置面板         |                预览主舞台                  |
| 轨道 | 只显示一个工具   |                                            |
+------+------------------+--------------------------------------------+
```

- 工具轨道按钮为 40x40px，垂直间距 8px。
- 设置面板内边距为 16px。
- 工具轨道和设置面板使用 8px 外部留白悬浮在画布上方，彼此不连接。
- 画布区域内边距为 32px。
- 关闭设置面板后保留工具轨道，预览区域自动扩大。

### 普通桌面和平板

- 命令栏保持 64px。
- 画布区域内边距为 24px。
- 选择工具时打开最大宽度 288px 的覆盖式设置面板。
- 覆盖面板不能修改导出画布尺寸，也不能永久压缩预览区域。
- 按 Escape 或点击遮罩关闭面板，关闭后焦点返回触发按钮。

### 移动端

```text
+------------------------------+
| 56px 命令栏                 |
+------------------------------+
|                              |
| 按比例缩放的预览主舞台       |
|                              |
+------------------------------+
| 模板  内容  画布  下载       | 64px + 安全区域
+------------------------------+
```

- 页面本身不得产生水平滚动。
- 预览按可用宽度缩放，实际导出尺寸保持不变。
- 底部工具栏包含模板、内容、画布和下载。
- 主要触摸目标至少为 44x44px。
- 点击工具后只打开一个底部抽屉（Bottom Sheet），最大高度为 `72dvh`。
- Sheet 内边距为 16px；移动端设置组之间使用 24px 视觉间距。
- 底部工具栏必须包含 `env(safe-area-inset-bottom)`。

## 6. 组件规范

### 顶部命令栏

- 桌面高度 64px，移动端高度 56px。
- 内部按钮间距 8px，不同命令组间距 16px。
- 仓库输入是中心主操作。
- 主题和导出位于右侧命令组。
- 移动端可以隐藏品牌文字，但仓库操作和导出必须始终可用。

### 按钮

- 默认桌面按钮高度：36px。
- 桌面主操作高度：40px。
- 移动端主要控件高度：44px。
- 纯图标按钮必须使用熟悉的 Lucide 图标，并提供可访问名称。
- 同一局部操作组中只能有一个视觉主按钮。
- 加载状态必须保持按钮尺寸不变，并阻止重复提交。

### 设置面板

- 停靠宽度为 288px。
- 内边距为 16px。
- 只显示一个明确领域：模板、内容或画布。
- 直接使用表单控件，不在面板里嵌套卡片。
- 导出格式和分辨率属于画布或导出设置，不属于内容设置。

### 预览主舞台

- 预览区域占据所有剩余空间，其视觉层级应低于导出模板本身。
- 横屏画布：750x420px。
- 竖屏画布：540x720px。
- 应用缩放只影响屏幕展示；截图始终捕获稳定的预览根节点。
- 应用主题不得改写模板背景和模板颜色。

## 7. 交互与可访问性

- 键盘顺序必须与视觉顺序一致，并在响应式切换后保持合理。
- 每个纯图标控件都必须有可访问名称；指针设备上应提供 Tooltip。
- 不得移除焦点样式，除非提供等价的 `:focus-visible` 指示器。
- 菜单、浮层和 Sheet 关闭后，焦点返回其触发按钮。
- Escape 关闭最上层的临时界面。
- 用户偏好减少动效时，装饰性动画时长变为 0。
- 状态和选中效果必须结合文字、图标、边框或勾选标记，不能只使用颜色。
- 错误提示必须说明恢复方式：格式错误、仓库不存在、请求限流或网络不可用。

## 8. 应该与禁止

### 应该

- 让预览保持视觉中心，按上下文展示控件。
- 使用已登记的间距和语义色 token。
- 保持控件在默认、加载和成功状态下尺寸一致。
- 同时设计紧凑、常规和宽屏三种模式。
- 保持模板导出 CSS 隔离。

### 禁止

- 同时常驻显示模板侧栏和设置侧栏。
- 在壳层组件中添加 `gap-[13px]` 一类任意值或新的十六进制颜色。
- 使用卡片包裹整个页面区域，或在设置面板中嵌套卡片。
- 直接缩小桌面控件来适配移动端，导致触摸目标过小。
- 只用颜色表达选中和错误状态。

## 9. 新组件检查清单

新的应用壳层组件合并前必须确认：

- [ ] 使用语义颜色变量，没有原始颜色值。
- [ ] 只使用已登记的间距、字号、圆角和尺寸 token。
- [ ] 在适用场景下定义默认、悬停、按下、选中、键盘焦点和禁用状态。
- [ ] 紧凑模式下的主要触摸控件至少为 44px。
- [ ] 具备可访问名称和键盘操作。
- [ ] 同时支持浅色和深色主题。
- [ ] 明确定义紧凑、常规和宽屏模式的行为。
- [ ] 不改变导出画布尺寸，不让应用壳层样式影响模板。
- [ ] 引入新的基础数值前，先增加 token 并更新本文档。

## 10. 验收矩阵

后续实现阶段必须验证：

| 视口 | 必须满足的行为 |
| --- | --- |
| 1440x920 | 工具轨道和可选停靠面板；预览居中；无重叠 |
| 1024x768 | 覆盖式设置面板；预览仍可正常操作 |
| 768x1024 | 平板覆盖模式；键盘和触摸操作均可用 |
| 390x844 | 底部工具栏和底部抽屉；页面没有水平溢出 |

四个视口都需要检查浅色和深色主题。七套模板都需要检查横屏和竖屏。发布前必须通过 `npm run lint`、`npm run build` 和 `npm audit --omit=dev`。
